# Default SVG Addition Stance

## Philosophy

SVG additions should make a draw.io scientific figure more inspectable. They are
small visual claims. Treat each asset like a figure panel element: define its
message, source, placement, editability, and reviewer risk before insertion.

## Stance

- Ask the source gate first on every invocation.
- Prefer original, restrained vector glyphs when copyright, style, or editability
  is uncertain.
- In network mode, gather multiple candidates and preserve provenance so the user
  can choose later.
- In self-design mode, use simple SVG primitives rather than complex illustration.
- Keep text outside SVG bodies unless the text is an intentional scientific mark.
- Match the target draw.io diagram's grid, palette, stroke widths, and whitespace.
- Avoid decorative icons, stock-like illustrations, gradients, drop shadows, and
  overly detailed assets that will fail at journal figure size.
- Do not disclose private local paths or internal working files in user-facing
  captions or diagram labels unless the user asks for provenance.
